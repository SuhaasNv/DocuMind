## Description

Backend for the Insight Garden RAG system: auth, document upload/lifecycle, RAG (chunking, embedding, pgvector), BullMQ processing, and chat (including SSE streaming).

**Stack:** NestJS, Prisma 7, PostgreSQL (pgvector), Redis (BullMQ), JWT.

## Project setup

1. **Start infrastructure** (from repo root): `docker compose up -d`  
   - PostgreSQL (pgvector) on 5432, Redis on 6379. Credentials match `.env.example` (user/password, db `insight_garden`).

2. Copy `.env.example` to `.env`. **Required:** `DATABASE_URL` (use `localhost:5432` when Postgres is in Docker), `JWT_SECRET`. **For BullMQ:** `REDIS_HOST=localhost`, `REDIS_PORT=6379`. See [docs/INCIDENT-AUTH-FIX.md](docs/INCIDENT-AUTH-FIX.md) for root cause, .env checklist, and sanity checklist.

3. **Migrations and Prisma client:**

```bash
npm install
npx prisma migrate dev   # applies migrations; generate runs automatically
```

4. **Start the server:**

```bash
npm run dev
# or: npm run start:dev
```

If the database is unreachable, the app exits with an error (no silent catch).

**Troubleshooting:** If Migrate says "Can't reach database server at localhost:51213" (or similar), your `DATABASE_URL` is using the wrong port. Docker Compose Postgres is on **port 5432**. In `backend/.env` set:
`DATABASE_URL="postgresql://user:password@localhost:5432/insight_garden"`
Then ensure Postgres is running: from repo root run `docker compose up -d`.

**API base:** `http://localhost:3000` (or `PORT` from env).

- **Auth:** `POST /auth/register`, `POST /auth/login` → `{ user, accessToken }`. Use `Authorization: Bearer <accessToken>` on protected routes.
- **Documents:** `POST /documents/upload` (multipart `file`), `GET /documents`, `GET /documents/:id`, `DELETE /documents/:id`, `POST /documents/:id/retry`, `GET /documents/:id/retrieval`, `POST /documents/:id/chat`, `POST /documents/:id/chat/stream` (SSE).
